GitHub Pages: Root vs Docs

Entenda as duas formas de publicar seu site no GitHub

Opção 1: Pasta Root (/)

Seus arquivos ficam na raiz do repositório.

Como configurar:

  • Seu arquivo principal deve se chamar index.html
  • Coloque todos os arquivos na raiz do repositório
  • Vá em Settings → Pages → Branch: main → Save

Vantagens:

  • Mais simples para projetos pequenos
  • Todos os arquivos visíveis na raiz
  • Fácil de configurar

Desvantagens:

  • Pode poluir o repositório
  • Mistura código e arquivos públicos
# Estrutura do repositório:
scriptflow-platform.github.io/
├── index.html
├── style.css
├── script.js
└── README.md

Opção 2: Pasta /docs

Seus arquivos ficam em uma pasta chamada "docs" na raiz.

Como configurar:

  • Crie uma pasta chamada docs no repositório
  • Coloque todos os arquivos do site dentro dela
  • Vá em Settings → Pages → Branch: main → /docs → Save

Vantagens:

  • Mantém o repositório organizado
  • Separa código e arquivos públicos
  • Permite versionamento do site separadamente

Desvantagens:

  • Configuração um pouco mais complexa
  • URLs ficam mais longas (ex: seu-site.com/docs)
# Estrutura do repositório:
scriptflow-platform.github.io/
├── docs/
│   ├── index.html
│   ├── style.css
│   └── script.js
├── src/
└── README.md

✅ Recomendação para Iniciantes

Use a pasta Root (/) para projetos simples como o ScriptFlow.

É mais fácil de configurar e manter para sites menores.
